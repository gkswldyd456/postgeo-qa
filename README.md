| 버전 | URL |
|------|-----|
| 운영 (QA팀) | https://gkswldyd456.github.io/postgeo-qa/ |
| Dev 미리보기 | https://gkswldyd456.github.io/postgeo-qa/index-dev.html |

## 대시보드 기능

| 기능 | 설명 |
|------|------|
| 상태 뱃지 클릭 | 웹이식 / QA 상태 순환 변경 |
| 💾 저장 | GitHub에 변경사항 저장 (병합 저장 — 동시 작업 안전) |
| 🔄 변경 감지 배너 | 다른 편집자가 저장하면 20초 내 알림 |
| 필터 (행 1) | 카테고리 · 웹이식 · QA 상태별 필터링 |
| 필터 (행 2) | devStatus 드롭다운 · 이슈# 검색 · PR# 검색 · 🔔 미확인 토글 · ✅ QA재확인 토글 |
| 진행률 바 | 개발자 처리 현황 (미확인/처리중/보류/완료 비율) |
| 컬럼 리사이즈 | 헤더 경계 드래그로 너비 조정 |
| 컬럼 가이드 ⓘ | 각 컬럼 헤더 클릭 시 체크 기준 팝오버 |
| 편집 모드 ✏ | 항목 이름 수정·삭제 / 카테고리 추가·수정·삭제 (삭제 시 항목 전체 삭제 경고) |
| 드래그앤드롭 (항목) | 편집 모드 ⠿ → 같은 카테고리 내 순서 변경 또는 다른 카테고리로 이동 (ID 자동 변경) |
| 드래그앤드롭 (카테고리) | 편집 모드 카테고리 헤더 ⠿ → 카테고리 순서 변경 |
| 인라인 삽입 | 편집 모드에서 행 사이 hover → ＋ 위치 추가 |
| 이슈/메모 모달 | 카드형 메모 목록 — 메모별 텍스트·이슈#·PR#·devStatus·댓글 스레드 |
| 댓글 스레드 | 메모별 QA↔개발자 양방향 댓글 (role 배지 표시) |
| 역할 설정 | ⚙ 설정 → QA / 개발자 선택 (댓글 작성 시 역할 표시) |
| 전체 코멘트 탭 | 특정 기능과 무관한 전반적 의견 기록 + PR·devStatus·댓글 지원 |
| 코멘트 타입 관리 | 설정에서 타입 추가/삭제 |

## 상태값 순환

| 컬럼 | 순환 |
|------|------|
| 웹이식 | `? 미확인` → `✓ 완료` → `△ 부분` → `✗ 미이식` |
| QA | `미테스트` → `통과` → `실패` → `건너뜀` |

### 개발자 처리 상태 (devStatus)

| 값 | 표시 | 의미 |
|----|------|------|
| null | 🔔 미확인 | 개발자 아직 미확인 |
| pending | ⏳ 대기 | 확인함, 처리 대기중 |
| in-progress | 🔧 수정중 | 수정 진행중 |
| done | ✅ 완료 | 수정 완료 (QA 재확인 필요) |
| hold | ⏸ 보류 | 보류 |
| not-issue | — 해당없음 | 이슈 아님으로 판단 |

## 개발자 협업 흐름

1. **QA팀**: 메모 작성 (버그 내용, 이슈 번호, PR 번호 등)
2. **개발자**: devStatus 선택 + 댓글로 답변 (`?view=dev` 링크로 미확인 항목 바로 확인)
3. **QA팀**: 댓글로 추가 질문 또는 재확인 결과 기록
4. **devStatus: done** → QA팀이 ✅ QA재확인 필터로 모아보고 재테스트

## 편집 권한 설정

### 토큰 보유자 (직접 설정)
1. 대시보드 **⚙ 설정** → GitHub 사용자명 + Personal Access Token 입력
2. Fine-grained PAT 권장: `Contents: Read and write` 권한만 부여

### 비개발자 편집자 (공유 링크 방식)
GitHub 계정 없어도 아래 형식의 URL을 한 번 열면 자동으로 편집 권한 설정:

```
https://gkswldyd456.github.io/postgeo-qa/#ghowner=gkswldyd456&ghtoken=토큰값
```

최초 1회만 이 URL로 접속 → 이후엔 일반 URL로 접속해도 편집 가능.

## 동시 작업 안내

여러 명이 동시에 편집해도 자동으로 병합 저장됩니다.

- **서로 다른 항목** 편집 → 저장 시 자동 병합, 데이터 유실 없음
- **같은 항목** 편집 → `_modified` 타임스탬프 기준, 더 늦게 수정한 내용이 반영됨
- **devStatus** 충돌 → `devStatusAt` 타임스탬프 기준, 더 늦게 변경한 값 우선
- **댓글(comments[])** → id 기준 합집합 (양쪽 추가분 모두 보존, 로컬 삭제 반영)
- **카테고리·항목 순서** → 로컬 순서 우선, 원격에서만 추가된 항목은 원격 상대 위치 보존
- **변경 감지** → 다른 편집자가 저장하면 20초 내 상단 배너로 알림

> 같은 카테고리에서 동시에 항목 이동·삭제 작업은 피하는 것을 권장합니다.
